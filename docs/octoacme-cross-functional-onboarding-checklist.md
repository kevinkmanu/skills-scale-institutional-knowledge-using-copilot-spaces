# OctoAcme — Cross-Functional Onboarding Checklist

## Purpose
Ensure every new team member — regardless of their role — has the context and access they need to contribute effectively from day one.

---

## For All Roles

- [ ] Introductory meeting scheduled with PM and Product Manager
- [ ] Added to the project communication channels (Slack/Teams, mailing lists)
- [ ] Granted access to the project board (e.g., GitHub Projects)
- [ ] Given access to the project repository and relevant documentation
- [ ] Reviewed the [Project Management Overview](./octoacme-project-management-overview.md)
- [ ] Reviewed the [Roles & Personas](./octoacme-roles-and-personas.md) doc and identified key collaborators
- [ ] Reviewed the [Project Initiation](./octoacme-project-initiation.md) doc (Project Charter / One-pager)
- [ ] Added to recurring team ceremonies (standup, planning, retrospective)
- [ ] Introduced to the team in standup or a dedicated intro meeting

---

## For Developers

- [ ] Local development environment set up and verified
- [ ] CI/CD pipeline access confirmed (able to trigger and view builds)
- [ ] Reviewed branching and PR conventions in the repository
- [ ] Paired with a team member on a first task to learn the workflow
- [ ] Reviewed the [Execution & Tracking](./octoacme-execution-and-tracking.md) doc

---

## For QA Lead

- [ ] Reviewed current test strategy and existing test suites
- [ ] Access confirmed to CI test pipeline and test reporting tools
- [ ] Met with Developers to discuss testability requirements
- [ ] Reviewed release checklist and quality gates with PM
- [ ] Reviewed the [Release & Deployment](./octoacme-release-and-deployment.md) doc

---

## For UX/UI Designer

- [ ] Access granted to design tools (e.g., Figma) and shared design library
- [ ] Reviewed existing wireframes, prototypes, and design system
- [ ] Met with Product Manager to understand current feature backlog and priorities
- [ ] Introduced to Developers for design handoff process walkthrough
- [ ] Reviewed any existing user research or usability findings

---

## For Business Analyst

- [ ] Reviewed existing requirements documentation and user stories
- [ ] Met with key stakeholders to understand business goals and priorities
- [ ] Access confirmed to requirements management and project tracking tools
- [ ] Established process for requirements sign-off with PM and Product Manager
- [ ] Reviewed the [Project Planning](./octoacme-project-planning.md) doc

---

## For Operations Lead

- [ ] Access confirmed to deployment pipelines, monitoring dashboards, and alerting tools
- [ ] Reviewed existing runbooks and incident response procedures
- [ ] Met with PM to review operational risks and current system health
- [ ] On-call rotation and escalation path established
- [ ] Reviewed the [Release & Deployment](./octoacme-release-and-deployment.md) doc

---

## For Customer Success Manager

- [ ] Met with Product Manager to review current customer commitments and launch timelines
- [ ] Access confirmed to customer health tracking and feedback tools
- [ ] Reviewed release notes and training documentation from the previous release
- [ ] Introduced to Developers and PM for release coordination process
- [ ] Customer pilot and rollout plan reviewed with PM

---

## Manager / PM Responsibilities

- [ ] All system access provisioned before start date
- [ ] Role added to the project's RACI/Interaction Matrix in the Roles & Personas doc
- [ ] 30/60/90-day expectations documented and shared with new team member
- [ ] First retrospective check-in scheduled (30 days post-start)
